有Composition API后，有些场景可以不需要pinia

kitty2024-07-142025-04-06

是看了前端欧阳的文章 复制下来 略微修改+添加说明

前言

有些业务场景功能很复杂，如果将所有代码都写在一个vue组件中，vue文件的代码量可能就几千行了，维护极其困难。

这时我们就需要将其拆分为多个组件，拆完组件后就需要在不同组件间共享数据和业务逻辑。

有的小伙伴会选择将数据和业务逻辑都放到<font style="color:rgb(71, 193, 168);">pinia</font>中，这样虽然可以解决问题。

但是如果将所有的复杂的业务都放在<font style="color:rgb(71, 193, 168);">pinia</font>中，那么<font style="color:rgb(71, 193, 168);">pinia</font>就会变得很乱。

什么是Composition API?

组合式 API (Composition API) 是一系列 API 的集合，使我们可以使用函数而不是声明选项的方式书写 Vue 组件。是有一个概括性的术语。例如ref()、reactive()等。

什么是hooks?

就是钩子函数。。。说中文会死。。。

将数据和业务逻辑都封装到**hooks**中

这时你还有另外一个选择，使用<font style="color:rgb(71, 193, 168);">Composition API</font>将数据和业务逻辑都抽取到<font style="color:rgb(71, 193, 168);">hooks</font>中。

<font style="color:rgb(71, 193, 168);">state</font>状态的定义和更新以及具体的业务逻辑全部由<font style="color:rgb(71, 193, 168);">hooks</font>内部维护，组件只负责使用<font style="color:rgb(71, 193, 168);">hooks</font>暴露出的<font style="color:rgb(71, 193, 168);">state</font>状态和方法。

下面是我们封装的<font style="color:rgb(71, 193, 168);">hooks</font>：

组件只需要使用hooks中暴露出的状态<font style="color:rgb(71, 193, 168);">count</font>和<font style="color:rgb(71, 193, 168);">doubleCount</font>，以及方法<font style="color:rgb(71, 193, 168);">increment</font>和<font style="color:rgb(71, 193, 168);">decrement</font>，无需关注具体的内部逻辑是如何实现的。

上面的封装其实是有问题的，如果我们将组件拆为两个，分别为<font style="color:rgb(71, 193, 168);">CountValue.vue</font>（显示<font style="color:rgb(71, 193, 168);">count</font>的值）和<font style="color:rgb(71, 193, 168);">CountBtn.vue</font>（修改count变量值）。

由于我们的<font style="color:rgb(71, 193, 168);">count</font>变量是在<font style="color:rgb(71, 193, 168);">useStore</font>函数中定义的，所以每调用一次<font style="color:rgb(71, 193, 168);">useStore</font>函数都会重新定义一个<font style="color:rgb(71, 193, 168);">count</font>变量。

在我们这里<font style="color:rgb(71, 193, 168);">CountValue</font>和<font style="color:rgb(71, 193, 168);">CountBtn</font>组件都在<font style="color:rgb(71, 193, 168);">setup</font>中调用了<font style="color:rgb(71, 193, 168);">useStore</font>函数，通过<font style="color:rgb(71, 193, 168);">useStore</font>函数拿到的就不是同一个<font style="color:rgb(71, 193, 168);">count</font>变量。

这样就会导致我们在<font style="color:rgb(71, 193, 168);">CountBtn</font>中修改了<font style="color:rgb(71, 193, 168);">count</font>变量的值，但是<font style="color:rgb(71, 193, 168);">CountValue</font>组件中显示的<font style="color:rgb(71, 193, 168);">count</font>变量的值一直没变。

多个组件同时调用**hooks**如何共享同一份state状态

问题的原因是因为每次调用<font style="color:rgb(71, 193, 168);">useStore</font>函数都会生成一个新的<font style="color:rgb(71, 193, 168);">count</font>变量。就不将<font style="color:rgb(71, 193, 168);">count</font>变量的定义写在<font style="color:rgb(71, 193, 168);">useStore</font>函数中，将变量的定义写在<font style="color:rgb(71, 193, 168);">useStore</font>函数的外面就可以了。

总结

在多个组件中需要复用状态和业务逻辑的情况时，我们可以不将这些状态和业务逻辑写到<font style="color:rgb(71, 193, 168);">pinia</font>中，而是使用<font style="color:rgb(71, 193, 168);">Composition API</font>将状态和业务逻辑封装成一个<font style="color:rgb(71, 193, 168);">hooks</font>。为了多个组件同时调用<font style="color:rgb(71, 193, 168);">hooks</font>时能够共用同一个<font style="color:rgb(71, 193, 168);">state</font>状态，我们需要将定义的变量写在<font style="color:rgb(71, 193, 168);">useStore</font>函数外面。
